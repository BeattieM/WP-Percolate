Complete Percolate Settings

Percolate Settings Page â€¹ Percolate Demo â€” WordPress Percolate User Type - There are two user types the plugin is designed for; Single Percolate Users and Multiple Percolate Users. If there is more than one Percolate user, use the Multiple Users option. Percolate will provide this info.

Percolate API Key - The API Key authenticates your account and allows Wordpress to communicate with Percolate via the API. Percolate will provide this info. Website - This specifies which Percolate dot com posts will be imported. There will typically be only one option, select it. Publish Status - This setting dictates whether Percolate posts will be imported as draft or live posts. We always recommend that posts be imported as drafts so they can be previewed and finalized before publishing to your Wordpress theme. Default License Author - When a Wordpress author does not exist for a Percolate license author, posts from Percolate will be assigned from this Wordpress user. (i.e. if a new Percolate user creates a post but has not been mapped to a WP user) License Authors - Allows you to map the Percolate license author with the Wordpress Author. There can be a unique Wordpress author for every Percolate license author publishing to WP. Click ''refresh'' to update WP with any new Percolate users that need to be mapped to a WP author. Otherwise the default in the step above will be used. Category - This setting will assign a WordPress Category to all of the posts from Percolate (ex: "blog"). This can dictate where posts from Percolate will appear. Post Type - This setting allows you to specify what WordPress post type content from Percolate will be imported as. In most cases the default type ''post'' should be used. Save - click 'Save Changes ' to ensure settings are updated.
